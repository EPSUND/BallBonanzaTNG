# Ball Bonanza (BallBonanzaTNG)

En remake i React av Java-spelet [EPSUND/BallBonanza](https://github.com/EPSUND/BallBonanza)
("Kul med kulor"), byggd med samma teknikstack som
[WordOnWordNext](https://github.com/EPSUND/WordOnWordNext).

## Spelet

- 10×10-bräde med kulor i sex färger.
- Klicka på en kula och sedan på en tom ruta – kulan rullar dit om det finns en fri väg.
- Minst **fem kulor i samma färg** i rad (vågrätt, lodrätt eller diagonalt) rensas och ger poäng.
- Varje drag som inte ger poäng lägger ut tre nya kulor ("Nästa" visar färgerna).
- Spelet är över när brädet är fullt.

Poäng: 5/6/7/8/9 kulor ger 1/3/10/15/25 poäng (diagonal rad +1), plus 1 bonuspoäng
per extra rad som rensas samtidigt – samma system som i originalet.

## Teknik

- **React 18 + TypeScript + Vite**, byggs till en statisk sajt på **GitHub Pages**.
- **Supabase** (REST) för den globala topplistan. Inget eget backend.
- Ljudeffekterna är originalets wav-filer, uppspelade via WebAudio.

## Kommandon

```
npm install          # installera beroenden
npm run dev          # dev-server med HMR (Vite)
npm run build        # tsc -b && vite build → dist/
npm run typecheck    # bara TypeScript, ingen emit
npm run preview      # servera dist/ lokalt (under rätt base-path)
```

## Engångsinstallation av topplistan (Supabase)

Topplistan använder samma Supabase-projekt som Ord på Ord men en egen tabell
`bb_scores`. Kör en gång i Supabase **SQL Editor**:

```sql
create table public.bb_scores (
  id bigint generated by default as identity primary key,
  name text not null,
  score integer not null,
  balls_cleared integer not null,
  created_at timestamptz not null default now()
);

alter table public.bb_scores enable row level security;

-- Anon får läsa och lägga till, men aldrig ändra eller radera.
create policy "bb_scores_select" on public.bb_scores
  for select to anon using (true);
create policy "bb_scores_insert" on public.bb_scores
  for insert to anon with check (true);
```

## Deploy

GitHub Actions bygger och deployar `dist/` till GitHub Pages vid push till `main`
(`.github/workflows/deploy.yml`). Engångsinställning i GitHub:
**Settings ▸ Pages ▸ Source ▸ GitHub Actions**.
